HTML4 페이지구조

HTML4에서 페이지 구조는 div, span태그를 이용, 구역을 설정하고 id 값을 부여하고 구분함.
*단점: 1. 사용자마다 id 값을 다르게 부여하여 구조 이해 / 유지보수에 어려움
2. 검색엔진이 웹문서를 확인할 때도 메뉴, 본문 구별 어려움

HTML5 페이지 구조

시맨틱 태그를 사용함
1. 특정 기능에 맞는 태그를 사용하여 페이지 구조를 구분
2. 페이지 구조를 쉽게 파악할 수 있고, 정확한 정보를 검색할 수 있음

*header : 특정 부분의 머릿말로 주로 검색어form나 메뉴nav를 넣음
- nav : 다른 사이트나 페이지로 이동하는 태그를 모아놓은 태그. 특정 태그에 종속되지 않고 어느 곳에서나 사용할 수 있음

*main : 핵심 콘텐츠를 담는 태그. 웹 문서마다 다르게 보여주는 내용으로 구성, 웹문서에서 한번만 사용

*section : 웹 문서에 컨텐츠가 들어가는 영역, 컨텐츠를 주제별로 묶을 때 사용, section 태그 안에 section 태그 사용 가능

-article : 독립적인 콘텐츠를 담는 태그. 여러개 사용 가능 / section 태그 넣기 가능 웹 페이지의 내용이 들어가는 영역. 이 영역은 다른 곳으로 배포하거나 재사용 가능

*aside : '사이드바'라 불리며 본문 외에 기타 내용을 담고있는 영역. 주로 광고/링크모음 등표현

*footer : 웹페이지의 맨 아래쪽. 회사 소개/저작권/연락처 등의 정보 표시

*address : 주소 표시할때 사용
